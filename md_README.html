AES Encryptor (Multithreaded Benchmark)

Проект являє собою застосунок для паралельного мультипоточного та послідовного шифрування за алгоритмом AES-128 (в режимі CTR). Проект чітко поділений на дві основні частини: алгоритмічну бібліотеку на чистому C++17 (STL) та графічний інтерфейс користувача (GUI) на основі фреймворку Qt6.

Для перевірки коректності алгоритму використовується Google Test (GTest).

‍🤖 Примітка: Документація та Doxygen коментарі в коді були створені за допомогою штучного інтелекту Google Gemini.

Технології

  • Мова: C++17 (використання STL, std::thread, std::async, std::future)
  • GUI: Qt6 (Qt Widgets)
  • Збірка: CMake
  • Тестування: Google Test
  • Документація: Doxygen

Структура проекту

  • src/core/ — "Ядро" шифрування. Бібліотека, повністю незалежна від Qt. Включає класифікацію перетворень блоку AES та реалізацію патернів проектування для перемикання між послідовною та паралельною логікою.
  • src/gui/ — Виконуваний файл GUI. Відповідає за введення ключа, вибір файлів для шифрування/дешифрування, відображення процесу та результатів бенчмарку.
  • src/benchmark/ — Модуль для вимірювання часу виконання та порівняння швидкодії алгоритмів на різних розмірах даних.
  • tests/ — Юніт-тести GTest, що перевіряють коректність AES за стандартами NIST та ідентичність результатів багатопоточного та однопоточного режимів.
  • tools/ — Допоміжні bash-скрипти для автоматизованої збірки та запуску.

Реалізовані алгоритми

  1. AES Block Core: Базові перетворення SubBytes, ShiftRows, MixColumns, AddRoundKey та розширення ключа KeyExpansion.
  2. Sequential AES (CTR Mode): Однопотокове виконання шифрування у режимі лічильника.
  3. Parallel AES (CTR Mode): Багатопотокове виконання шифрування, що динамічно розподіляє блоки даних між потоками за допомогою std::async та пулу потоків на основі std::thread::hardware_concurrency().

Патерни проектування (GoF)

Проект було спеціально спроектовано з активним застосуванням патернів для досягнення максимальної гнучкості та розширюваності коду:

  1. Strategy (Стратегія) Використовується для інкапсуляції різних методів шифрування за допомогою загального інтерфейсу IEncryptor. Дозволяє легко замінювати послідовну реалізацію (AesSequential) на паралельну (AesParallel) прямо під час виконання без зміни клієнтського коду бенчмарку чи GUI.
  2. Factory Method (Фабричний метод) Клас EncryptorFactory використовується для інстанціювання об'єктів алгоритмів шифрування за їхнім типом (EncryptorType::Sequential або EncryptorType::Parallel), приховуючи складну логіку створення та ініціалізації від головного вікна.

Збірка та запуск

Клонуйте репозиторій, після чого перейдіть у корінь проекту:

git clone https://github.com/wwerniss/aes-encryptor.git
cd aes-encryptor

Для вашої зручності у репозиторії передбачені готові скрипти для збірки та запуску проекту під Unix-системи у директорії tools/.

‍Примітка: На вашій платформі повинні бути встановлені CMake, C++17 компілятор (GCC/Clang/MSVC) та бібліотека Qt6 (віджети). Фреймворк GTest завантажиться автоматично під час збірки завдяки CMake FetchContent.

Linux / macOS (через скрипти)

Для Unix-подібних систем існують розроблені автоматичні bash-скрипти у папці tools/.

# Зібрати весь проект
./tools/build.sh
# Запустити автоматичні тести GTest
./tools/tests.sh
# Запустити застосунок з GUI після збірки
./tools/run.sh

Будь-яка платформа (вручну через CMake)

# 1. Створення директорії та генерація Makefile
mkdir build && cd build
cmake ..
# 2. Збірка проекту
make -j4 # або cmake --build .
# 3. Запуск застосунку
./AesEncryptorApp
# 4. Запуск автоматичних тестів
ctest --output-on-failure
